🟢 SPRINT 1 — Foundation
🔷 EPIC: Identity & Profile
🧩 User Story: Register account
Description:
As a user, I want to create an account, so that I can access the platform.

🧩 User Story: Log in
Description:
As a user, I want to log into my account, so that I can continue using the platform.

🧩 User Story: Manage profile
Description:
As a user, I want to view and edit my profile, so that others can understand who I am and what I can do.

🧩 User Story: View other user profile
Description:
As a user, I want to view another user’s profile, so that I can evaluate whether to interact with them.

🔷 EPIC: Task Lifecycle (Part 1)
🧩 User Story: Create task
Description:
As a user, I want to create a task, so that I can request help from others.

🧩 User Story: View my created tasks
Description:
As a user, I want to see the tasks I have created, so that I can manage them.

🧩 User Story: Browse tasks
Description:
As a user, I want to browse available tasks, so that I can find tasks to respond to.

🧩 User Story: View task details
Description:
As a user, I want to view detailed information about a task, so that I can decide whether to participate.

🧩 User Story: View my active tasks
Description:
As a user, I want to see tasks I am currently involved in, so that I can track ongoing interactions.

🟡 SPRINT 2 — Matching
🔷 EPIC: Responses
🧩 User Story: Respond to task
Description:
As a user, I want to respond to a task, so that I can offer my help.

🧩 User Story: View my responses
Description:
As a user, I want to see the tasks I have responded to, so that I can track my activity.

🧩 User Story: See response status
Description:
As a user, I want to see the status of my responses, so that I understand what is happening with them.

🔷 EPIC: Selection & Match
🧩 User Story: View responses to my task
Description:
As a user, I want to see users who responded to my task, so that I can choose one.

🧩 User Story: Review responses
Description:
As a user, I want to review responses one by one, so that I can evaluate candidates efficiently.

🧩 User Story: Select response
Description:
As a user, I want to select one response, so that the task can move to the next stage.

🧩 User Story: Create match
Description:
As a user, I want a match to be created after selecting a response, so that we can start working together.

🧩 User Story: View task status
Description:
As a user, I want to clearly see the current status of a task, so that I understand its state.

🟠 SPRINT 3 — Communication
🔷 EPIC: Chat
🧩 User Story: Chat after match
Description:
As a user involved in a matched task, I want to chat with the other participant, so that we can coordinate task execution.

🧩 User Story: Restrict chat to matched users
Description:
As a user, I want chat access to be limited only to matched interactions, so that communication remains relevant and safe.

🧩 User Story: View conversation list
Description:
As a user, I want to see my active conversations, so that I can navigate between them.

🟠 SPRINT 4 — Execution & Completion
🔷 EPIC: Task Execution
🧩 User Story: Mark task as in progress
Description:
As a user involved in a matched task, I want to see when a task is in progress, so that I understand its current phase.

🧩 User Story: Initiate task completion
Description:
As a user, I want to initiate task completion, so that the interaction can be finalized.

🧩 User Story: Confirm task completion
Description:
As a user, I want to confirm task completion initiated by the other participant, so that completion is mutual.

🧩 User Story: Raise completion concern
Description:
As a user, I want to raise a concern about task completion, so that issues can be addressed.

🔷 EPIC: Feedback
🧩 User Story: Leave feedback
Description:
As a user, I want to leave feedback after a completed task, so that others can evaluate participants.

🧩 User Story: View feedback
Description:
As a user, I want to see feedback on other users, so that I can decide whether to interact with them.

🔷 EPIC: Moderation
🧩 User Story: Report user
Description:
As a user, I want to report another user or interaction, so that unsafe behavior can be handled.

🧩 User Story: Handle reports
Description:
As an admin, I want to review and act on reports, so that the platform remains safe.

🟣 SPRINT 5 — Compensation
🔷 EPIC: Payments (MVP)
🧩 User Story: Specify compensation in task
Description:
As a user, I want to specify compensation when creating a task, so that others understand the reward.

🧩 User Story: View compensation details
Description:
As a user, I want to see compensation details in a task, so that I can decide whether to participate.

🧩 User Story: Confirm compensation after completion
Description:
As a user, I want to confirm that compensation is fulfilled after task completion, so that the interaction is fairly closed.
